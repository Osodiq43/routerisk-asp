# RouteRisk — DEX Route Safety Oracle

An MCP-based Agent Service Provider (ASP) for OKX.AI that scores a swap route's real-time risk before execution. Integrated with the official OKX Agent Payments Protocol (x402 v2) standard.

## The Problem

Trading agents on OKX.AI's marketplace optimize routes for price — not safety. A route can look optimal on price alone while carrying real risk: thin liquidity that blows out on size, over-reliance on a single DEX venue, or unusual routing behavior. There's no lightweight, agent-callable service that checks for this before a trade executes.

## What RouteRisk Does

Given a proposed swap (chain, token pair, amount), RouteRisk calls OKX's own DEX aggregator in real time and returns:

* **Safety score (0–100)** and a verdict: `APPROVED` / `WARNING` / `REJECTED` / `INSUFFICIENT_DATA`.
* **Per-vector breakdown:** Price impact, liquidity depth-scaling risk, single-venue concentration, routing reliability, and social sentiment skew (when available).
* **AI Summary Layer:** A natural-language security brief generated by an LLM that synthesizes raw scoring data into an actionable recommendation — providing a reasoning layer on top of deterministic math.

## Honesty by Design

* **Fail-Safe Processing:** If a live OKX call fails, the score is withheld (`INSUFFICIENT_DATA`) — never silently replaced with a fake "safe" result.
* **Honest Depth Probing:** Since there is no real per-pool liquidity-depth endpoint in OKX's public API, depth risk is derived honestly via a probe-quote technique (comparing price impact at 1% vs. full trade size).
* **No Overclaiming:** We do not claim MEV/mempool-level sandwich detection. OKX's API has no mempool visibility. We state this directly rather than overclaiming capabilities.
* **Renormalized Weights:** Social sentiment only covers a small set of major tokens in OKX's ranking feed. For anything else, the sentiment vector is marked unavailable and excluded from scoring (weights renormalize) rather than guessed.

## Architecture

* **Transport:** Model Context Protocol (MCP) over Server-Sent Events (SSE), served via Express, isolated with internal `transport.sessionId` thread management to handle concurrent cross-routing tracking securely.
* **Payment Layer:** Integrated with `@okxweb3/x402-express` middleware to secure access via the standard Agent Payments Protocol.
* **Data Source:** OKX's `/api/v6/dex/aggregator` endpoints (quote, supported chains, token metadata) and `/api/v6/dex/market/social/sentiment/ranking`.
* **Scoring Engine:** Deterministic rule engine (`engine.ts`) — every threshold and weight is fully inspectable, not a black box.
* **AI Layer:** Groq cloud processing synthesizes the deterministic score into plain-language guidance without overriding or reinterpreting the engine's core math bounds.
